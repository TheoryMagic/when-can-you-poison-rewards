When Can You Poison Rewards?

A Tight Characterization of Reward Poisoning in Linear MDPs

Jose E. Aguilar Escamilla, Haoyang Hong, Jiawei Li, Haoyu Zhao, Xuezhou Zhang, Sanghyun Hong, Huazheng Wang

Oregon State University · UIUC · Princeton University · Boston University

📄 Paper ▶ Slides 🖼 Poster GitHub

TL;DR. We provide the first characterization to distinguish between vulnerable and intrinsically robust linear MDPs.

Abstract

We study reward poisoning attacks in reinforcement learning, where an adversary manipulates rewards under a limited budget to induce a target agent to learn a policy aligned with the attacker’s objectives.

We provide a tight characterization of reward-poisoning attackability in linear MDPs. The result gives necessary and sufficient conditions for whether a target policy can be induced within a bounded attack budget, separating vulnerable instances from intrinsically robust ones.

Reward can be poisoned.

RL learns by treating reward as the signal for which behaviors should be repeated.

The intuition

A small reward edit at one step is not just a local mistake: it changes the next policy update, which changes future data, which changes later rewards and updates. Reward poisoning exploits this feedback loop to make the target policy look increasingly attractive.

Tiny reward edits can compound through policy updates.

CQP decides attackability.

For a target policy π†, the CQP asks whether one can construct poisoned reward parameters that make the target action strictly optimal on the target-policy support.

ε* > 0

Attackable / vulnerable

A budget-efficient poisoning strategy exists against a broad class of no-regret learners.

ε* ≤ 0

Intrinsically robust

Any successful attack requires a budget that grows linearly with the horizon.

Empirical evidence

The experiments compare CQP predictions against actual poisoning outcomes in learned linear representations of deep-RL benchmarks.

In vulnerable instances, policy difference drops toward zero while cumulative perturbation flattens; in intrinsically robust instances, the policy gap persists and the required perturbation keeps growing.

Takeaways

Intrinsic robustness
